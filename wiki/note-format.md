# 笔记格式规范

AI 整理后的笔记遵循以下规范，确保与 Obsidian 完全兼容。

## YAML Front-matter

每篇笔记头部必须包含元数据：

```yaml
---
title: 笔记标题
created: 2026-06-07
source: "B站 小王Albert → BV1xxxxxx"
tags:
  - 标签1
  - 标签2
rating: ⭐⭐⭐⭐⭐
topics:
  - 核心主题1
  - 核心主题2
---
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | ✅ | 笔记标题，与文件名保持一致 |
| `created` | ✅ | ISO 日期格式 `YYYY-MM-DD` |
| `source` | ✅ | 内容来源，如 `"平台 UP主 → 链接"` |
| `tags` | ✅ | YAML 列表格式的标签，至少 2 个 |
| `rating` | ❌ | 推荐指数，1~5 星 |
| `topics` | ❌ | 核心主题列表，用于快速检索 |

## 笔记结构

```markdown
---
tags: [标签1, 标签2]
created: YYYY-MM-DD
source: "来源"
---

# 笔记标题

> 一句话摘要（核心观点）

---

## 📌 核心观点

- 观点 1
- 观点 2

## 📂 详细内容

按逻辑分块，使用二级/三级标题。

## 💡 我的思考

个人感悟、联想、批判性思考。

## 🔗 关联笔记

- [[其他笔记标题]]  — 关联原因

## 📎 附件

原始链接或其他资源。
```

## 分类规则

| 内容类型 | Obsidian 目录 |
|---------|-------------|
| AI / 机器学习 / LLM | `Notes/AI学习/` |
| 编程 / 开发 / 框架 | `Notes/编程开发/` |
| 经济学 / 商科 / 金融 | `Notes/经济学/` |
| 国际政治 / 时评 | `Notes/日常杂谈/` |
| 语言学习 / 雅思 | `Notes/语言学习/` |
| 学术 / 论文解读 | `Notes/学术研究/` |
| Vlog / 生活分享 | `Notes/日常杂谈/` |
| 其他 | `Notes/未分类/` |

## 双向链接规范

- 使用 `[[笔记标题]]` 格式
- 关联笔记放在 `## 🔗 关联笔记` 段落
- 尽量跨分类链接，丰富知识图谱
